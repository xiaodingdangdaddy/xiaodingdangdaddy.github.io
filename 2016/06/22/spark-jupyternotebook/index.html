安装使用jupyter notebook 进行Spark开发

2016-06-22

在spark学习过程中，看到一些国外的开发者使用Jupyter notebook进行spark开发。可以像写文章一样，一边调试一边记录，还能保存为markdown格式，随手就是一个详实的博客，确实挺吸引人。
但最终让我入坑的，还是使用了IBM的data science workbench，第一次发现，写代码也可以这样干。这种感觉就像拿了一个笔记本，一边写思路，一边写代码，写完就能调试运行出结果，全程都在一个浏览器上完成。

———————华丽的分割线——————–
先吐个槽：国内貌似用这个的不多，教程也是残次不齐，在各种baidu、google和stackOverFlow翻滚后，总结下自己的部署过程

前提

我是在IP尾数为61(master)和62(slave)的两台Vmware centos虚拟机上，部署了Hadoop,yarn和spark.
调试运行均正常。
这部分网上的教程非常多，不再详述

安装jupyter notebook

运行 pip install jupyter 直接傻瓜式安装。
如果你和我一样是安装在centos7上，很可能遇到gcc编译错误。解决方法可以参考我另一篇文章 ：gcc error
