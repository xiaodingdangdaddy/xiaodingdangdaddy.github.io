安装使用jupyter notebook 进行Spark开发

2016-06-22

在spark学习过程中，看到一些国外的开发者使用Jupyter notebook进行spark开发。可以像写文章一样，一边调试一边记录，还能保存为markdown格式，随手就是一个详实的博客，确实挺吸引人。
但最终让我入坑的，还是使用了IBM的data science workbench，第一次发现，写代码也可以这样干。这种感觉就像拿了一个笔记本，一边写思路，一边写代码，写完就能调试运行出结果，全程都在一个浏览器上完成。

配置客户端浏览器登陆

centos7上，上述安装完成后，在运行jupyter notebook启动后，很可能出现warning: No web browser found: could not locate runnable browser
这是由于默认安装后，只能在本地浏览器访问。
像我这样的，安装在虚拟机中，然后想在win10浏览器里访问，还要进行一下内容

  • 启动ipython，$ ipython

  • 创建远程连接密码，In [1]: from notebook.auth import passwd; passwd()
    输入两次密码，将得到一个字符串，比如’sha1:d0270e88f33b:212384922803ca9bd49d1fb6c11ab42651ccb358’，该终端不要关掉，复制该字符串，后面会用到；

  • 生成jupyter的配置文件，$ jupyter notebook –generate-config，将会在home目录下生成一个隐藏文件夹.jupyter，该文件夹中有一个jupyter的配置文件；

  • 打开配置文件，$ vim ~/.jupyter/jupyter_notebook_config.py，复制以下内容粘贴到配置文件中
    c.NotebookApp.ip = ‘*’
    c.NotebookApp.password = u’ sha1:d0270e88f33b:212384922803ca9bd49d1fb6c11ab42651ccb358’#这里需要改为第（2）步你自己电脑上生成的密钥字符串
    c.NotebookApp.port = 9001
    c.InteractiveShellApp.matplotlib = ‘inline’

  • 设置jupyter notebook在后台不间断运行，且配置成错误信息输出到屏幕（可选），$ nohup jupyter notebook >/dev/null 2>&1 &
