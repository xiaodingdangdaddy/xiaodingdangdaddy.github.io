webTestSumary

2015-11-09

概要

web测试通常分为6个部分：

  • 1.功能测试
  • 2.性能测试
  • 3.用户界面测试
  • 4.兼容性测试
  • 5.安全测试
  • 6.接口测试

1.功能测试

1.1链接测试

测试所有链接是否按指示的那样链接到正确的页面
测试所链接的页面是否存在
测试是否有孤立页面，也就是服务器上有页面，但界面上没有链接指向该页面，只有知道正确的URL地址，才能访问
note:
推荐两种自动化测试网站链接的软件：
Xenu Link Sleuth
HTML Link Validator

1.2表单、数据校验测试

测试提交按钮能否正常工作
测试提交后能否返回正确的提示信息
测试后台能否检查提交表单的完整性，是否确项目未填就提交
测试提交信息的正确性，比如用户填写工作地是浙江省，但城市填写了北京

1.3 Cookies测试

测试cookies是否起作用
测试cookies是否对信息加密

1.4数据库测试

测试事务的原子性，既一整个事务中间某部出错，是否全部重置

1.5其他针对页面功能的测试

根据页面功能不同，设计其他测试项

2.性能测试

2.1响应时间测试

从用户提交操作，到返回页面结果，如果响应时间过长，如超过5秒，用户就会离开

2.2负载测试

测试最大并发用户数，系统能允许多少个用户同时在线
测试最佳并发用户数。同一时刻最多允许多少个用户同时访问

2.3压力测试

测试长时间允许，系统的状态

3.用户界面测试

3.1导航测试

测试导航是否直观，易于操作
测试导航、菜单、页面结构等风格是否一致

3.2图形测试

测试页面中显示的图片是否都有明确目的
测试页面字体风格是否一致
测试背景和前景颜色是否一致
测试图片的大小和质量

3.3内容测试

测试页面信息的正确性，比如价格标注是否有误，页面显示内容是否正确

3.4整体界面测试

页面是否给用户一个整体感
用户能否凭直觉找到需要的信息在哪里

4.兼容性测试

4.1平台测试

系统能否适应不同的操作系统，如linux windows mac

4.2浏览器测试

页面在不同浏览器中的显示和内容加载是否正常

4.3分辨率测试

页面在不同分辨率模式下，显示是否正常，字体是否正确，文本能否对齐等

5.安全测试

5.1目录设置

测试后台每个目录下，都有index.html 或main.html页面，这样就不会显示该目录下所有内容

5.2 SSL测试

如果页面使用SSL，那测试是否有替代的页面，以免老的浏览器不支持

5.3登录测试

测试非法的用户名\口令是否能登录
测试是否有登录错误次数限制
测试是否可以不登录就访问系统某个页面
测试是否有超时限制，比如登录后15分钟内没有任何操作，是否需要重新登录才可用

5.4测试mysql注入

5.5测试跨站脚本攻击
